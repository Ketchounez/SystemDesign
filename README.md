Вариант: **13 — Система бронирования отелей**

# 1. Изучение текста задания

Рассматривается система бронирования отелей. Пользователь может искать отели, просматривать информацию о них и создавать бронирования.

Основные сущности системы:

* пользователь
* отель
* бронирование

Через API система должна обеспечивать следующие возможности:

* регистрация пользователей
* поиск пользователей
* создание и просмотр отелей
* создание бронирований
* просмотр бронирований пользователя
* отмена бронирования

---

# 2. Роли пользователей и внешние системы

## Пользователь

Основной пользователь системы. Работает с системой через веб-приложение.

Доступные действия:

* регистрация в системе
* поиск и просмотр отелей
* создание бронирования
* просмотр своих бронирований
* отмена бронирования

---

## Внешняя система

### Email Service

Сервис отправки email-уведомлений.

Используется системой для отправки:

* подтверждения бронирования
* уведомления об отмене бронирования

---

# 3. Описание softwareSystem и диаграмма System Context

Основная система:

**Hotel Booking System**

Система взаимодействует со следующими участниками:

**User**

Использует систему для поиска отелей и создания бронирований.

**Email Service**

Используется для отправки уведомлений пользователям.

Диаграмма **System Context (C1)** показывает систему, пользователя и внешний сервис отправки уведомлений.

---

# 4. Основные задачи пользователей (Use Cases)

## Работа с пользователями

Создание пользователя

```
POST /users
```

Получение пользователя по логину

```
GET /users/login/{login}
```

Поиск пользователей

```
GET /users/search?name=&surname=
```

---

## Работа с отелями

Создание отеля

```
POST /hotels
```

Получение списка отелей

```
GET /hotels
```

Поиск отелей по городу

```
GET /hotels?city=
```

---

## Работа с бронированиями

Создание бронирования

```
POST /bookings
```

Получение бронирований пользователя

```
GET /users/{id}/bookings
```

Отмена бронирования

```
DELETE /bookings/{id}
```

# 5. Контейнеры системы

Система состоит из следующих контейнеров.

## Web Application

Веб-приложение, через которое пользователь взаимодействует с системой.

Функции:

* отображение интерфейса
* отправка HTTP-запросов к backend API

Технология:

React

---

## API Application

Основной backend системы.

Функции:

* обработка HTTP-запросов
* работа с пользователями
* работа с отелями
* обработка операций бронирования

Технология:

Java / Spring Boot

---

## Booking Service

Сервис, отвечающий за операции бронирования.

Функции:

* создание бронирования
* отмена бронирования
* получение списка бронирований пользователя

Технология:

Java REST Service

---

## Database

Хранилище данных системы.

Основные таблицы:

* users
* hotels
* bookings

Технология:

PostgreSQL

---

# 6. Взаимодействие контейнеров

User → Web Application
Пользователь работает с системой через веб-интерфейс.

Web Application → API Application
Фронтенд отправляет HTTP-запросы к backend.

API Application → Booking Service
API использует сервис бронирований для обработки операций бронирования.

API Application → Database
Backend сохраняет и получает данные из базы данных.

Booking Service → Database
Сервис бронирований сохраняет данные о бронировании.

Booking Service → Email Service
После создания или отмены бронирования пользователю отправляется email уведомление.
